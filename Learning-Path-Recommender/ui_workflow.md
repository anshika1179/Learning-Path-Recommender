# UI Design & Workflow

## UI Overview

The interface is designed to be simple, modern, and conversational while providing strong visual insights into learning progress.

Primary design goals:

* Minimal clutter
* Clear progress visualization
* Interactive conversational experience

---

## Step-by-Step User Workflow

### Step 1 — Onboarding

User provides:

* Current skills
* Learning goals
* Available weekly hours
* Preferred learning style
* Preferred platforms

Optional:

* Upload completed course history.

---

### Step 2 — AI Analysis Phase

System:

1. Embeds learner profile
2. Searches relevant courses
3. Detects skill gaps
4. Builds initial roadmap

Loading indicators show AI reasoning progress.

---

### Step 3 — Dashboard View

User sees:

* Timeline of recommended courses
* Estimated completion dates
* Course details and alternatives
* Overall progress indicators

---

### Step 4 — Weekly Plan Generator

System converts roadmap into actionable tasks:

Example:

Week 1:

* Course X module 1–3
* Watch recommended playlist
* Practice exercises

---

### Step 5 — Conversational Interaction

User can:

* Adjust time availability
* Request easier/harder courses
* Ask explanations
* Get quizzes or summaries

---

### Step 6 — Continuous Adaptation

When progress updates:

* System re-analyzes profile
* Adjusts future recommendations
* Updates timelines automatically

---

## Additional UI Enhancements (Added)

* Dark/light theme toggle
* Skill progress heatmap
* AI confidence indicator for recommendations
* Bookmark/favorite courses feature
* Notification reminders for weekly goals

---

## UX Goals

* Motivation through visibility
* Flexibility without confusion
* Continuous guidance from AI
